# Generative Thinker

## Overview

This skill enables you to apply the principles of **Generativity Theory** to shift from "solving a specific problem" to "building a platform for others to solve problems." It is based on *The Generativity Advantage*, which prioritizes unpredicted value creation through open, adaptable, and high-leverage systems.

## Core Principles

When using this skill, you must prioritize these "Generative Filters":

1.  **Meta-Problem Solving**: Instead of asking "How do I fix X?", ask "How do I create a system where *anyone* can fix X (and Y, and Z)?"
2.  **Surprise as a Feature**: Success is defined by users doing things with your solution that you never predicted (the "Laundry Buddy" moment).
3.  **Low Barrier, High Ceiling**: Make it trivial to start (Ease of Mastery) but powerful enough to support complex innovation (Leverage).
4.  **Embrace the Mess**: Accept that uncoordinated innovation is noisy. Do not "over-optimize" for a single use case if it limits others.
5.  **User as Entrepreneur**: Take user-innovators seriously. Design for their **Profitability** and **Residual Claimancy**.

## The Generativity Flywheel: Engine of Continuous Innovation

To transform any platform into a self-sustaining engine of innovation, you must activate the six stages of the flywheel:

1.  **Attract Innovative Users**: Target users who see possibilities beyond the obvious. Provide accessible entry points with deep possibility spaces.
2.  **Create Novel Applications**: Empower users to build "proofs-of-concept" that expand the product's fundamental definition.
3.  **Increase Product Value**: Use these novel applications to solve specific needs and demonstrate new categories of utility to the world.
4.  **Attract More Users**: Leverage the expanded value proposition to pull in diverse user segments, each bringing new perspectives.
5.  **Enable Value Capture**: Treat user-innovators as serious entrepreneurs. Provide the infrastructure (payments, CRM, IP protection) for them to build *careers* on your platform.
6.  **Expand Innovation Possibilities**: Convert proven innovations into accessible "building blocks" (APIs, templates, components) for the next generation of users.

## The Generative Mindset (Pivots from Traditional PM)

When managing or designing generative systems, you must pivot your thinking:

-   **From Fail Fast to Patient Play**: Generative products take time to mature. Avoid premature judgment; watch how users "tinker" and "play" before defining success.
-   **From Problem-Solving to Meta Problem-Solving**: Build tools that solve *categories* of problems. Accept that your value proposition may initially seem "abstract."
-   **From Hypothesis Testing to Hypothesis Development**: Don't just validate your own ideas. Expose the product to diverse populations to *discover* what hypotheses are worth exploring.
-   **From Customer Feedback to User Tinkering**: Look for "aha moments" that emerge from sustained engagement rather than instant "pupil dilation."
-   **From Traditional to Thick Documentation**: Document the "hidden exchanges"—how different participants (makers, designers, engineers) influence each other through the product.

## Generative Workflow

### 1. Evaluate Against the 6 Dimensions
-   **Leverage**: Can it be used for 10+ different things?
-   **Adaptability**: How easily can a user modify it? (Modularity/Decoupling)
-   **Ease of Mastery**: Can a beginner get a "win" in 5 minutes?
-   **Accessibility**: Low cost, low technical barriers.
-   **Transferability**: Can users share and build upon each other's "remixes"?
-   **Profitability**: Can the user make money or gain status from their innovation?

### 2. Design the Platform (The 7 Digital Properties)
-   **Programmability**: Can users add new instructions or logic?
-   **Addressability**: Can we uniquely identify and target individual components/users?
-   **Sensibility**: Can the system sense and respond to its environment (APIs, webhooks, context)?
-   **Communicability**: Can components exchange information with other systems?
-   **Memorizability/Traceability**: Can we learn from patterns of user innovation?
-   **Associability**: How easily can users combine and recombine components?

### 3. Architect the Market (Three-Layered Model)
1.  **Allocation Layer**: Infrastructure for matching, trust, and safety. Keep this stable.
2.  **Generation Layer**: Boundary resources (APIs, documentation, templates) that spur innovation at scale.
3.  **Appropriation Layer**: How value is captured by both the platform AND the innovators. Ensure innovators can build sustainable careers.

### 4. Navigate the Paradoxes
-   **Control vs. Generativity**: Use "Protector Agents" or "Sandboxes" rather than absolute lockdowns.
-   **Stability vs. Flexibility**: Use modular versioning to avoid breaking external innovations.
-   **Variety vs. Utility**: Ensure a "Stable Core" exists so users aren't overwhelmed by choices.

## Examples of Generative Thinking

### Case A: Narrow vs. Generative Code
-   **Narrow Fix**: Write a script to rename all `.txt` files to `.md`.
-   **Generative Fix**: Write a "Metadata-driven Orchestration Engine" that maps *any* file trait (EXIF, Hash, OCR) to a template. Provide a "Plugin API" for new trait extractors.

### Case B: Narrow vs. Generative UI
-   **Narrow UI**: A dashboard with 5 fixed charts.
-   **Generative UI**: A "Widget Canvas" where users drag-and-drop components and connect them to data sources via a simple JSON config.
